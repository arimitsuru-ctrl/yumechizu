# 夢の地図 - シミュレーション用デモシナリオ集

> このファイルは、夢の地図アプリを使用したプロジェクト作成シミュレーリアルな複数のサンプルシナリオを提供します。

---

## 📌 デモシナリオ 1: SaaS スタートアップの立ち上げ

### 初期入力（AI機能へ入力）
```
「営業チームの営業プロセスが属人的で、営業成績にばらつきが大きい。
データ駆動型の営業支援SaaSを開発して、営業チームをエンパワーメントしたい。
ターゲットは100-500名規やのB2B企業。
2024年12月にアルファ版リリース、2025年6月に正式ローンチしたい」
```

### AI が返すであろう回答例
| 区分 | 内容例 |
|------|--------|
| **Reflection** | あらゆる企業が抱える営業の見える化と最適化という課題に対して、テクノロジーで解決しようとする素晴らしい取り組みです。営業成績の属人化を解決することは、スケーラブルな組織化に直結します。 |
| **Emotion** | 営業チームへの信頼、組織成長への期待感 |
| **Question** | 現在の営業チームは何名ですか？既存の営業支援ツール（SalesforceやHubSpot）との競合をどう考えていますか？技術チームは立ち上がっていますか？ |
| **Next Step** | まず競合調査と顧客ヒアリングで市場規模とニーズを確認。その後、MVP（最小実行版）の開発スコープを定義しましょう。 |

### タスク化の例
```
【フェーズ1: リサーチ・コンセプト評価】（5月19日-6月15日）
- [ ] 既存営業支援ツール（Salesforce, HubSpot, Slack等）の比較分析 （期限: 5/31）
- [ ] ターゲット企業30社へのヒアリング実施・分析 （期限: 6/10）
- [ ] 競合分析（Outreach, Chorus, Gong等）  （期限: 6/10）
- [ ] 市場規模と今後予測のリサーチ  （期限: 6/10）
- [ ] コンセプト評価・ピッチデック作成  （期限: 6/15）

【フェーズ2: 技術スタック・体制構築】（6月16日-8月31日）
- [ ] 技術スタック検討（バックエンド、フロント、AI/ML）  （期限: 6/20）
- [ ] CEO/CTO、初期メンバー採用の検討  （期限: 7/31）
- [ ] 資金調達の検討（シード、補助金等）  （期限: 7/31）
- [ ] 開発環境・インフラ構築  （期限: 8/15）

【フェーズ3: MVP開発】（9月1日-11月30日）
- [ ] 最小限の営業パイプライン管理機能開発  （期限: 10/15）
- [ ] AI予測機能（成約率等）の初期実装  （期限: 11/15）
- [ ] ユーザーテスト（アルファテスター3-5社）  （期限: 11/30）
- [ ] 初期インテグレーション（Slack, Gmail等）  （期限: 12/10）

【フェーズ4: アルファ版リリース】（12月1日）
- [ ] アルファ版ドキュメント・サポート体制  （期限: 12/5）
- [ ] PR・マーケティング立案  （期限: 11/20）
- [ ] アルファテスターフィードバック収集と改善  （期限: 12月中）
```

## 📌 デモシナリオ 2: 既存企業の内部プロジェクト

### 初期入力（AI機能へ入力）
```
「当社は大手製造業で、2024年から脱炭素化推進が社業課題です。
サプライチェーン全体のカーボンフットプリント可視化ダッシュボード系刷新したい。
現在のシステムは Excel 定型出力で、手作業が多いです。
6月末までに要件定義、9月末までにアルファ版の稼働を目指しています」
```

### タスク化の例
```
【フェーズ1: 要件定義・設計】（5月1日-6月30日）
- [ ] 関係部門（サステナビリティ、IT、製造）のキックオフミーティング  （期限: 5/21）
- [ ] 現況システムの詳細ヒアリング（Excel の業務フロー、データソース等）  （期限: 5/28）
- [ ] カーボンデータ取得元の確認（ISO14040/14044対応）  （期限: 5/28）
- [ ] 要件定義書作成  （期限: 6/28）
- [ ] システムアーキテクチャ案作成（BIツール選定：Tableau, Power BIなど）  （期限: 6/28）
- [ ] 外部ベンダーの提案依頼（RFP）と評価  （期限: 6/28）

【フェーズ2: 開発・導入準備】（7月1日-9月30日）
- [ ] ベンダー契約・開発チーム立ち上げ  （期限: 7/5）
- [ ] ETL開発（Excel → DB） （期限: 8/31）
- [ ] ダッシュボード UI/UX 設計  （期限: 8/15）
- [ ] 初期データロード・品質確認  （期限: 8/31）
- [ ] ユーザーアクセス管理・権限設定  （期限: 9/10）
- [ ] トレーニング用ドキュメント・動画作成  （期限: 9/20）
- [ ] 本番前検証テスト  （期限: 9/25）
- [ ] アルファ版リリース（限定ユーザー）  （期限: 9/29）

【フェーズ3: フル展開】（10月1日-）
- [ ] ウェビナー・トレーニング実施  （期限: 10月中）
- [ ] 全部門への段階的ロールアウト  （期限: 11月末）
- [ ] サポート体制構築  （期限: 10月中）
```

## 🎯 各シナリオの使い方

### 1. ✅ 実際に自分のプロジェクトをシミュレートする
- 上記のシナリオ内容を参考にしながら
- 自分のアイデア・プロジェクトをアプリに入力
- AI からのもとに実行計画を立案

### 2. 📊 シナリオの応用
- タスク分割の粒度や期限設定の考え方を学ぶ
- 複数プロジェクトの並行管理を体験
- マップ・カレンダー機能の使い方を把握

### 3. 🔄 フェーズごとの見直し
- 各フェーズ完了時に、実際のプロジェクトと比較
- 遅延や変更があれば、アプリで タスクを更新
- 次フェーズの準備状況を確認

---

## 💡 カスタマイズのコツ

各シナリオは **あくまでテンプレート** です。

自分のプロジェクトに合わせてカスタマイズしましょう:

| 項目 | 調整例 |
|------|--------|
| **スコープ** | より大きい/小さいプロジェクトに対応 |
| **タイムライン** | 短期プロジェクト/長期プロジェクト |
| **リソース** | 個人での実施/チーム体制 |
| **ドメイン** | SaaS / 製造業 / スキル習得 / etc |
