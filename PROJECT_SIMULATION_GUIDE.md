# 🗺️ 夢の地図 - プロジェクト作成シミュレーションガイド

## 概要

このガイドは、「夢の地図」アプリを使用して、実際に新しいプロジェクトを企画・設計するためのスムーズな流れを提供します。AIを活用しながら、夢やアイデアをプロジェクト化していきます。

---

## 🚀 セットアップ

### 開発サーバーの起動

```bash
cd /home/arisawa/yumechizu/yumechizu-pages
wrangler pages dev --local
```

ブラウザで [http://localhost:8788](http://localhost:8788) にアクセスします。

## 📋 シミュレーション実行フロー

### ステップ 1: 夢・アイデアを入力する
**セクション:** ホーム > 🤖 AI機能 > 夢スケッチ

1. **「🤖 AI機能」カードをクリック**
   - アプリの AI チャットインターフェースが開きます
   
2. **夢やアイデアをテキストで入力**
   ```
   例：
   「社内コミュニケーション効率化のため、AIを使った自動翻訳システムを開発したい」
   「Webアプリで名刺交換機能を持つビジネスネットワークを作りたい」
   「企業のESG施策を可視化するダッシュボードを構築したい」
   ```

3. **エンターキーまたは送信ボタンで提出**

### ステップ 2: AIからの洞察を受け取る
**AI が以下を返します:**

| 項目 | 説明 |
|-----|-----|
| **Reflection（洞察）** | あなたのアイデアの本質や価値を分析 |
| **Emotion（感情）** | プロジェクトに関連する感情的な背景 |
| **Question（質問）** | さらに考えを深めるための質問 |
| **Next Step（次のステップ）** | 実行に向けた具体的なアクション |

### ステップ 3: タスク化する
**セクション:** タスク

1. **アイデアを具体的なタスクに分解**
   - 手伝ってもらう機能でAIにタスク案を生成してもらう
   - 手動でタスクを追加することも可能

2. **各タスクに:を設定**
   - 📅 期限日：実施期限を設定
   - 🔁 繰り返し：定期的なタスクの場合は設定

3. **カテゴリーで整理**
   （例：💼 仕事、🌟 夢、など）

### ステップ 4: スケジュール管理
**セクション:** 📅 カレンダー

1. **カレンダービューで進捗を可視化**
2. **日程の競合をチェック**
3. **必要に応じてタスクをドラッグ&ドロップで移動**

### ステップ 5: 進捗追跡
**セクション:** 🗺️ マップ

- プロジェクト全体の進捗を可視化
- 完了したプロジェクトはアーカイブセクションに自動移動
- プロジェクト間の関連性をマップで確認

## 💡 実践的なシミュレーションシナリオ例

### シナリオ A: スタートアップの新サービス企画

**初期アイデア:**
```
「リモートワーカー向けの、チーム内のコミュニケーションギャップを埋めるAIチャットボットサービスを作りたい」
```

**期待される AI フロー:**
1. Reflection: このサービスが解決する根本的な問題を洞察
2. Emotion: チームの絆、生産性への思い
3. Question: 「ターゲット企業規模は？」「どのコミュニケーション課題が最も深刻？」
4. Next Step: 「まずサーベイを実施してターゲット検証」

**その後のタスク化:**
```
- タスク: ターゲット企業へのヒアリング調査
- タスク: 競合サービス分析
- タスク: 技術スタック検討
- タスク: 初期プロトタイプ開発計画
```

### シナリオ B: 企内プロジェクトの最適化

**初期アイデア:**
```
「既存の販売管理システムを、AIを使ってセールスパイプライン自動化に変えたい」
```

**期待される AI フロー:**
1. Reflection: 現在の課題と改善のメリットを分析
2. Emotion: 営業チームの負担軽減への期待
3. Question: 「導入までのタイムライン？」「既存システムとの互換性？」
4. Next Step: 「スコープ定義と利害関係者調整」

## 🎯 最適なシミュレーション体験のコツ

### 1. 詳細なアイデア入力
- より詳しく、背景も含めて入力するほど、AIの回答の質が向上します
- 「なぜこれが必要なのか？」も記述するとより効果的

### 2. AI との対話を繰り返す
- 最初の質問に対する AI の質問にさらに答える
- 複数回の対話でアイデアが洗練されます

### 3. タスク化して実行開始
- アイデア段階で満足せず、実行レベルまで落とす
- 実際のプロジェクトマネジメントで活用

### 4. 進捗の定期的なレビュー
- 毎週マップとカレンダーをレビュー
- 必要に応じてスケジュール調整

## 🔧 トラブルシューティング

### AI が応答しない場合
- **症状:** 「通信エラーが発生しました」と表示
- **原因:** Cloudflare Workers API エンドポイント（`yumechizu-ai.arimitsuru.workers.dev`）が応答していない可能性
- **解決:** ネットワーク接続を確認し、ブラウザのコンソールで詳細エラーを確認

### データが保存されない場合
- このアプリは LocalStorage を使用しています
- ブラウザのシークレットモードでは動作しありません
- キャッシュをクリアしてリロード

### UI が表示されない場合
- ブラウザをリロード（Ctrl+R または Cmd+R）
- Service Worker がキャッシュしている可能性がため、ハード更新（Ctrl+Shift+R）を試す

## 📊 シミュレーション後の活動

### 1. 結果のエクスポート
**セクション:** 📤 データ転送 > 📥 エクスポート
- データをJSONファイルとして保存
- チームメンバーとの共有に活用

### 2. チームでのの方法
- エクスポートされたファイルをチームで共有
- チェックボックスで責任者を割り当て
- 定期的に進捗をレビュー

### 3. 他のツールへの連携
- 保存したJSON を解析して、Slack や Notion などに送信可能
- プロジェクト管理ツール（Asana, Monday.com等）への自動生成タスク登録を検討

## 💬 よくある質問 (FAQ)

**Q: 複数のプロジェクトを同時にシミュレーションできますか？**
A: はい。タスク、マップ、カレンダーセクションで複数のプロジェクトを並行管理できます。

**Q: データは永続的に保存されますか？**
A: はい。LocalStorage に保存されます。ブラウザのキャッシュクリアで失われるため、定期的にエクスポートしてください。

**Q: AI機能の利用に制限はありますか？**
A: はい。1日5回まで使用可能です（「5/5」ボタンで確認可能）。

**Q: オフラインで使用できますか？**
A: はい。PWA として機能するため、一度ロードされたら オフラインでも基本機能は使用できます。ただし AI 機能はオンラインが必須です。
